## 一、实验目的

1. 理解第三方支付平台的基本架构与工作原理，掌握支付宝开放平台沙箱环境的使用方法；
2. 熟悉支付宝支付接口（当面付/条码支付）的调用流程，理解商户端、支付宝服务端、用户端的三方交互逻辑；

4. 能够独立完成沙箱环境下的支付下单、支付查询及退款等完整业务流程的联调测试；

## 二、实验原理与相关知识

### （一）支付宝支付流程概述

支付宝支付采用「请求-响应」模式，核心流程为：

```
商户系统生成订单 → 调用支付宝接口创建支付 → 用户确认支付 → 支付宝返回/通知支付结果 → 商户处理业务
```

### （二）沙箱环境机制

沙箱环境与生产环境完全隔离，拥有独立的数据体系。开发者无需签约即可调用接口。沙箱网关地址：

`https://openapi.alipaydev.com/gateway.do`

### （三）RSA2 非对称加密

采用 RSA2（SHA256withRSA）算法：商户私钥签名请求，支付宝公钥验签；支付宝私钥签名响应，商户公钥验签。

### （四）当面付条码支付

商户扫描用户付款码（auth_code），调用 `alipay.trade.pay` 接口，`scene` 设为 `bar_code`，同步返回支付结果。

## 四、实验步骤与操作过程

### 步骤一：注册并进入沙箱环境

1. 访问 [支付宝开放平台](https://open.alipay.com)，登录并完成实名认证；
2. 控制台 → 开发工具 → **沙箱**；
3. 记录沙箱 **APPID**（如 `9021000xxxxxxxxx`）；
4. 记录沙箱**买家账号**（用于登录沙箱 APP，默认密码 `111111`）。

### 步骤二：生成并配置 RSA2 密钥

1. 下载「支付宝开放平台密钥工具」；
2. 选择 PKCS1（非 JAVA）、RSA2，生成密钥对；
3. 将**应用公钥**粘贴到沙箱应用页面并保存；
4. 复制页面生成的**支付宝公钥**；
5. 将**应用私钥**保存至本地 `config.py`（保留 `-----BEGIN/END-----` 及换行）。

### 步骤三：安装依赖并配置项目

```powershell
cd alipay-sandbox-demo
python -m venv venv
.\venv\Scripts\activate
pip install -r requirements.txt
copy config.example.py config.py
# 编辑 config.py 填入 APPID 与密钥
```

### 步骤四：条码支付下单

1. 沙箱 APP 登录买家账号，打开「付款」获取 18 位付款码；
2. 修改 `pay_demo.py` 中 `AUTH_CODE` 为当前付款码；
3. 执行：

```powershell
python pay_demo.py
```

**实际支付成功返回：**

```json
  "total_amount": "0.01",
```

### 步骤五：订单查询与退款


使用沙箱买家账号登录 APP，实时刷新付款码后调用 `pay_demo.py`，对比 APP 账单与接口返回的 `trade_no`、金额是否一致。

## 五、实验结果与分析

### （一）支付下单结果

| 字段 | 实际值 | 含义 |
|------|--------|------|
| `code` | `10000` | 网关调用成功 |
| `out_trade_no` | `TEST20260626102906` | 商户侧唯一订单号 |
| `trade_no` | `2026062622001488880510034842` | 支付宝交易流水号 |
| `total_amount` | `0.01` | 订单总金额（元） |
| `receipt_amount` | `0.01` | 实收金额 |
| `buyer_logon_id` | `fja***@sandbox.com` | 付款用户 |
| `gmt_payment` | `2026-06-26 10:29:09` | 支付完成时间 |

**分析：** 条码支付为**同步接口**，`code=10000` 表示扣款成功。另有一笔较早订单 `TEST20260626102420` 因间隔过久查询时状态为 `TRADE_CLOSED`，导致退款失败（`ACQ.TRADE_HAS_CLOSE`），说明需在 `TRADE_SUCCESS` 状态下及时退款。

### （二）签名与验签验证

开启 `debug=True` 后，SDK 在控制台输出完整请求 URL 及 `sign` 参数。请求参数按字典序拼接后，使用商户**应用私钥** RSA2 签名；支付宝用对应公钥验签。响应中的 `sign` 字段由支付宝私钥生成，SDK 使用**支付宝公钥**自动验签，验签失败会抛出异常。

**结论：** 非对称加密保证请求参数在传输过程中无法被第三方篡改，是开放 API 安全的基础。

### （三）异常场景测试

| 场景 | 预期 sub_code | 实际现象 |
|------|---------------|----------|
| 无效付款码 | `ACQ.PAYMENT_AUTH_CODE_INVALID` | `code=40004`，`sub_code=ACQ.PAYMENT_AUTH_CODE_INVALID`，支付失败 |
| 订单已关闭后退款 | `ACQ.TRADE_HAS_CLOSE` | 订单 `TEST20260626102420` 状态为 `TRADE_CLOSED`，退款被拒 |
| 付款码过期 | `ACQ.PAYMENT_AUTH_CODE_INVALID` | 需重新在 APP 获取付款码 |
| 重复支付同一 out_trade_no | `ACQ.TRADE_HAS_SUCCESS` 或返回原交易 | 订单 `IDEM20260626103812` 第一次 `code=10000`；第二次相同订单号仍返回 `code=10000` 且 **trade_no 相同**（`2026062622001488880510032144`），未产生新交易，体现幂等 |

### （四）数据一致性验证

对比沙箱商家账号「账单明细」与接口返回：`trade_no`、`total_amount`、交易时间一致，说明沙箱账务与 API 数据同步正确。

### （五）订单查询、退款与退款查询结果

- **查询（退款前）：** `trade_status=TRADE_SUCCESS`，金额 0.01 元；
- **退款：** `code=10000`，`refund_fee=0.01`，退款请求号 `REFUND20260626102955`；
- **查询（退款后）：** `trade_status=TRADE_CLOSED`，`buyer_pay_amount=0.00`，表示全额退款完成；
- **退款记录查询：** 使用 `query_refund.py` 传入 `out_request_no` + `trade_no` 可查询退款详情。

## 六、实验总结与思考

### 1. 沙箱与生产环境区别

| 对比项 | 沙箱环境 | 生产环境 |
|--------|----------|----------|
| 网关 | `openapi.alipaydev.com` | `openapi.alipay.com` |
| APPID | 沙箱专用 | 正式应用 APPID |
| 资金 | 虚拟，无真实扣款 | 真实资金流转 |
| 签约 | 无需签约 | 需签约对应支付产品 |
| 用户 | 沙箱买家/卖家账号 | 真实支付宝用户 |

### 2. RSA2 签名防篡改作用

任何参数被修改都会导致签名不匹配，支付宝拒绝处理；商户验签响应可防止伪造的「支付成功」通知，形成双向信任。

### 3. 异步通知（notify_url）的重要性

网页支付等场景用户浏览器跳转不可靠，**异步通知**是支付宝主动 POST 到商户服务器的可靠通道。保障措施包括：HTTPS、验签、幂等处理（同一 `notify_id` 只处理一次）、主动查单补偿、返回 `success` 停止重试。

### 4. 迁移至生产环境额外步骤

- 创建正式应用并完成**当面付/电脑网站支付**等产品签约；
- 配置正式 RSA2 密钥与应用网关；
- `notify_url`、`return_url` 使用**备案域名 + HTTPS**；
- 关闭 `debug` 模式，切换正式网关；
- 接入风控、对账与监控告警。

## 七、附录：常见问题与解决方案

| 问题 | 解决方案 |
|------|----------|
| 无效的 AppID | 确认使用沙箱 APPID，非生产 APPID |
| 签名验证失败 | 检查私钥与上传公钥是否配对；保留 PKCS1 换行格式 |
| 付款码无效 | 付款码约 1 分钟有效，需从沙箱 APP 实时获取 |
| 收不到异步通知 | 当面付条码支付以同步结果为准；网页支付需公网 HTTPS 回调地址 |
| SDK 包名 | 实验代码使用 `python-alipay-sdk`（`from alipay import AliPay`） |

## 八、实验代码清单（核心片段）

**客户端初始化（alipay_client.py）：**

```python
from alipay import AliPay
from config import APP_ID, APP_PRIVATE_KEY, ALIPAY_PUBLIC_KEY, NOTIFY_URL

def create_alipay_client(*, debug: bool = False) -> AliPay:
    return AliPay(
        appid=APP_ID,
        app_notify_url=NOTIFY_URL,
        app_private_key_string=APP_PRIVATE_KEY,
        alipay_public_key_string=ALIPAY_PUBLIC_KEY,
        sign_type="RSA2",
        debug=debug,
    )
```

**条码支付（pay_demo.py）：**

```python
result = alipay.api_alipay_trade_pay(
    out_trade_no=out_trade_no,
    scene="bar_code",
    auth_code=AUTH_CODE,
    subject=subject,
    total_amount="0.01",
    timeout_express="2m",
)
```
